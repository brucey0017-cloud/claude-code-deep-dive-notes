# Engineering Patterns Summary

> 从 Claude Code 学到的工程模式

## 1. React 式设计
- **自定义 Reconciler**: 终端渲染
- **组件化**: 可复用的 UI 组件
- **状态管理**: 集中的状态存储
- **响应式更新**: 実时反映变化
## 2. 工具系统设计
- **统一接口**: 所有工具遵循相同协议
- **动态注册**: 运行时添加新工具
- **权限集成**: 工具级别的访问控制
- **错误处理**: 统一的错误处理机制
## 3. 多 Agent 架构
- **主从模式**: 主 Agent 分配任务
- **并行执行**: 多 Agent 同时工作
- **结果整合**: 绱一的输出合并
- **错误隔离**: 单个 Agent 失败不影响整体
## 4. 上下文管理
- **分层结构**: 系统/项目/会话/文件
- **智能压缩**: 保留关键信息
- **滑动窗口**: 最近 N 条消息
- **优先级队列**: 重要信息保留
## 5. 猦点恢复
- **自动重试**: 指数退避
- **降级处理**: 备用方案
- **状态保存**: 恢复后继续
- **错误日志**: 完整的错误追踪
## 6. 可观测性
- **日志系统**: 分级日志
- **状态监控**: 实时状态展示
- **性能指标**: 响应时间、Token 使用
- **错误追踪**: 完整的错误链
## 7. 扩展性
- **插件系统**: 标准化扩展
- **MCP 协议**: 跨平台兼容
- **配置驱动**: 灵活配置
- **热加载**: 无需重启更新
## 8. 用户体验
- **流式输出**: 实时反馈
- **进度展示**: 清晰的进度条
- **权限提示**: 明确的权限请求
- **错误友好**: 可理解的错误信息
